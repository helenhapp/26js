Події клавіатури на canvas

Мета заняття:
  • Навчитися працювати з елементом canvas за допомогою сучасного стандарту JavaScript (ES6+).
  • Зрозуміти принцип створення об'єктів за допомогою class (Об'єктно-орієнтоване програмування).
  • Реалізувати керування об'єктом (рух, зміна швидкості та розміру) за допомогою клавіатури.
  • Створити плавний цикл анімації за допомогою requestAnimationFrame.
✧ Підготовка: HTML та CSS
▶

Елемент <canvas> призначений для створення растрової графіки за допомогою JavaScript. Додай цей тег усередину тегу <body> твого HTML-файлу: 

<canvas id="canvas" width="500" height="500"></canvas>

Щоб полотно було чітко видимим і розташовувалося по центру вікна браузера, додай наступні базові стилі у свій CSS файл: 

body {
  display: flex;
  justify-content: center;
  align-items: center;
  height: 100vh;
  margin: 0;
  background-color: #f0f0f0;
}

canvas {
  background-color: white;
  border: 2px solid #333;
  box-shadow: 0 4px 10px rgba(0, 0, 0, 0.2);
}
✧ Крок 1: Ініціалізація Canvas та функція малювання
▶

Отримання контексту

Щоб малювати на полотні, необхідно отримати його контекст — спеціальний об'єкт, який містить усі методи для малювання плоских (2D) фігур.

[Image of HTML5 Canvas coordinate system] 
// 1. Знаходимо елемент полотна на сторінці
const canvas = document.getElementById("canvas");

// 2. Отримуємо 2D-контекст для доступу до методів малювання
const ctx = canvas.getContext("2d");

// 3. Зберігаємо розміри полотна у змінні для подальших розрахунків
const width = canvas.width;
const height = canvas.height;

Функція малювання кола

Оскільки ми будемо малювати коло багаторазово, доцільно винести цей процес в окрему функцію. Вона прийматиме параметри позиції, розміру та кольору. 

// Функція для малювання кола
const drawCircle = (x, y, radius, fillCircle, color) => {
  ctx.beginPath(); // Починаємо новий шлях малювання
  ctx.arc(x, y, radius, 0, Math.PI * 2, false); // Формуємо дугу на 360 градусів
  ctx.fillStyle = color; // Встановлюємо колір
  
  if (fillCircle) {
    ctx.fill(); // Заливаємо фігуру кольором
  } else {
    ctx.stroke(); // Малюємо лише контур
  }
};

🧪 Проміжне тестування

Перевіримо, чи коректно працює налаштований контекст та наша функція. Додай цей рядок внизу скрипта: 

drawCircle(width / 2, height / 2, 50, true, "red");

Після запуску коду ти маєш побачити червоне коло рівно по центру полотна. Якщо все працює успішно, цей тестовий рядок можна видалити.

✧ Крок 2: Створення Класу Ball
▶

Основи роботи з класами

У програмуванні Клас (Class) — це креслення або загальне правило для створення об'єктів. Щоб не прописувати властивості кожного м'яча окремо, ми описуємо їх лише один раз у класі, а браузер потім зможе створити за цим правилом скільки завгодно м'ячів.

Усередині класу є спеціальний блок — constructor(). Це своєрідні стартові налаштування. Він запускається автоматично щоразу, коли ми створюємо новий об'єкт, і одразу видає йому початкові характеристики: стартові координати, базову швидкість, розмір та колір. 

class Ball {
  constructor() {
    this.x = width / 2;   // Позиція по горизонталі
    this.y = height / 2;  // Позиція по вертикалі
    this.xSpeed = 5;      // Вектор швидкості по осі X
    this.ySpeed = 0;      // Вектор швидкості по осі Y
    this.radius = 10;     // Радіус об'єкта
    this.color = "black"; // Колір заливки
    this.speed = 5;       // Базовий модуль швидкості
  }

  // Метод для відображення об'єкта на полотні
  draw() {
    drawCircle(this.x, this.y, this.radius, true, this.color);
  }
}

🧪 Проміжне тестування

Щоб створити реальний об'єкт за шаблоном, потрібно використати ключове слово new. Напиши під закриваючою дужкою класу: 

const ball = new Ball(); // Створюємо екземпляр класу
ball.draw();             // Викликаємо метод малювання

На екрані має з'явитися чорний об'єкт. Це означає, що клас працює коректно.

Логіка руху та перевірка зіткнень

Важливо: наступний метод потрібно додати всередину класу Ball, одразу після методу draw().

Метод move() відповідає за рух об'єкта, постійно змінюючи його координати. Також він стежить, щоб об'єкт не вилетів за межі екрана: якщо м'яч торкається краю полотна, ми змінюємо його швидкість на протилежну (наприклад, з 5 на -5). Завдяки цьому математичному трюку м'яч "відбивається" від стінки і починає летіти в інший бік. 

  move() {
    // Оновлюємо координати
    this.x += this.xSpeed;
    this.y += this.ySpeed;

    // Перевірка зіткнення по осі X (ліва та права стінки)
    if (this.x - this.radius < 0 || this.x + this.radius > width) {
      this.xSpeed = -this.xSpeed;
    }

    // Перевірка зіткнення по осі Y (верхня та нижня стінки)
    if (this.y - this.radius < 0 || this.y + this.radius > height) {
      this.ySpeed = -this.ySpeed;
    }
  }
✧ Крок 3: Методи керування об'єктом
▶

Ці методи дозволяють нам віддавати команди об'єкту. Розміщуємо їх всередині класу Ball.

Керування напрямком

Метод setDirection() отримує команду — слово, яке вказує напрямок руху (наприклад, "up" або "right"). Щоб м'яч рухався чітко прямо і не летів по діагоналі, ми повинні правильно розподілити його швидкість. Тобто, якщо ми наказуємо м'ячу рухатись вгору, ми повністю зупиняємо його рух вліво чи вправо (ставимо xSpeed на 0), і направляємо всю його базову швидкість лише на рух вгору (у змінну ySpeed). 

  setDirection(direction) {
    if (direction === "up") {
      this.xSpeed = 0;              
      this.ySpeed = -this.speed;    // Рух вгору (з мінусом)
    } else if (direction === "down") {
      this.xSpeed = 0;
      this.ySpeed = this.speed;     // Рух вниз (додатній)
    } else if (direction === "left") {
      this.xSpeed = -this.speed;    // Рух вліво (з мінусом)
      this.ySpeed = 0;
    } else if (direction === "right") {
      this.xSpeed = this.speed;     // Рух вправо (додатній)
      this.ySpeed = 0;
    } else if (direction === "stop") {
      this.xSpeed = 0;              // Повна зупинка
      this.ySpeed = 0;
    }
  }

Зміна розміру та швидкості

Ці методи дозволяють змінювати властивості м'яча прямо під час гри. Ми використовуємо перевірку if, щоб м'яч не став занадто малим і не зник з екрана, а також щоб він не міг повністю зупинитися або почати рухатися у зворотному напрямку без команди. 

  changeSize(change) {
    // Додаємо передане значення change до поточного радіуса м'яча.
    // Якщо параметр change додатний (наприклад, 1), радіус збільшиться.
    // Якщо параметр change від'ємний (наприклад, -1), радіус зменшиться.
    this.radius += change; 

    // Захисна перевірка: щоб м'яч візуально не зник з екрана.
    // Якщо після обчислень радіус виявився меншим за 5 пікселів, 
    // ми примусово встановлюємо його на рівні 5.
    if (this.radius < 5) {
      this.radius = 5; 
    }
  }
changeSpeed(change) {
    this.speed += change;
    // Швидкість не може бути меншою за 1
    if (this.speed < 1) {
      this.speed = 1; 
    }

    // Оновлюємо поточну швидкість м'яча, щоб він одразу прискорився або сповільнився.
    
    // Перевіряємо горизонталь (вісь X): чи м'яч взагалі рухається вбік?
    if (this.xSpeed !== 0) {
      // Якщо швидкість більша за нуль, значить він летить вправо
      if (this.xSpeed > 0) {
        this.xSpeed = this.speed;  // Даємо нову швидкість (додатне число)
      } else {
        this.xSpeed = -this.speed; // Інакше він летить вліво (робимо число від'ємним)
      }
    }

    // Перевіряємо вертикаль (вісь Y): чи м'яч рухається вгору або вниз?
    if (this.ySpeed !== 0) {
      // Якщо швидкість більша за нуль, значить він летить вниз
      if (this.ySpeed > 0) {
        this.ySpeed = this.speed;  // Даємо нову швидкість (додатне число)
      } else {
        this.ySpeed = -this.speed; // Інакше він летить вгору (робимо число від'ємним)
      }
    }

🧪 Проміжне тестування

Давай перевіримо, чи вміє м'яч виконувати команди. У кінці файлу (після створення об'єкта) додай ці рядки: 

ball.changeSize(20);         // М'яч має стати більшим
ball.setDirection("right");  // М'яч має підготуватися до руху вправо
ball.move();                 // Робимо один крок
ball.draw();                 // Малюємо результат

Якщо м'яч на екрані збільшився і змістився вправо — методи працюють правильно. Після перевірки ці тестові рядки можна видалити.

✧ Крок 4: Обробка подій та Цикл анімації
▶

Увага: Весь код цієї частини ми пишемо після фігурної дужки, яка закриває клас Ball. Ми завершили створення "креслення" і тепер переходимо до керування програмою.

Словник команд

Створюємо об'єкт (словник), який зіставляє системні назви клавіш з нашими командами управління. 

const keyActions = {
  " ": "stop",              // Клавіша Пробіл
  "ArrowLeft": "left",      // Стрілка вліво
  "ArrowUp": "up",          // Стрілка вгору
  "ArrowRight": "right",    // Стрілка вправо
  "ArrowDown": "down",      // Стрілка вниз
  "z": "slowDown",          // Літера Z - зменшити швидкість
  "x": "speedUp",           // Літера X - збільшити швидкість
  "c": "decreaseSize",      // Літера C - зменшити розмір
  "v": "increaseSize"       // Літера V - збільшити розмір
};

Обробник події "keydown"

Метод addEventListener прослуховує натискання клавіш. Отримавши event.key, ми знаходимо дію у словнику та викликаємо необхідний метод. 

// Слухаємо натискання клавіш по всьому документу
document.addEventListener("keydown", (event) => {
  
  // Створюємо змінну, де будемо зберігати назву клавіші
  let key;

  // Перевіряємо довжину назви клавіші. 
  // У звичайних літер та цифр довжина назви дорівнює 1 (наприклад, "z", "x", "1").
  // У системних клавіш назва довга (наприклад, "ArrowUp", "Shift", "Escape").
  if (event.key.length === 1) {
    // Якщо це одиночний символ (літера), робимо його маленьким.
    // Це потрібно для того, щоб програма впізнала клавішу навіть при CapsLock.
    key = event.key.toLowerCase();
  } else {
    // Якщо це системна клавіша (наприклад, стрілка), залишаємо назву як є.
    key = event.key;
  }
  
  // Шукаємо отриману назву клавіші в нашому "словнику" команд
  const action = keyActions[key];
  
  // Якщо такої клавіші в словнику немає — нічого не робимо (виходимо з функції)
  if (!action) {
    return; 
  }

  // Перевіряємо, яку саме дію знайдено, і передаємо наказ м'ячу
  if (action === "slowDown") {
    ball.changeSpeed(-1); // Сповільнюємо
  } else if (action === "speedUp") {
    ball.changeSpeed(1);  // Прискорюємо
  } else if (action === "decreaseSize") {
    ball.changeSize(-1);  // Зменшуємо
  } else if (action === "increaseSize") {
    ball.changeSize(1);   // Збільшуємо
  } else {
    // Якщо це команда напрямку ("up", "down", "left", "right" або "stop")
    ball.setDirection(action); 
  }
});

Реалізація анімаційного циклу

Анімація створюється шляхом безперервного перемальовування полотна. Стандартом для цього є функція requestAnimationFrame. 

function animate() {
  // 1. Очищення полотна від попереднього кадру
  ctx.clearRect(0, 0, width, height);

  // 2. Відображення об'єкта
  ball.draw();
  
  // 3. Перерахунок координат для наступного кадру
  ball.move();

  // 4. Відображення зовнішньої рамки
  ctx.strokeRect(0, 0, width, height);

  // 5. Запит на виклик цієї ж функції для наступного кадру
  requestAnimationFrame(animate); 
}

// Запуск анімації
animate();
